# Beeber Boards & Cards

AI-powered platform for designing, balancing, and producing printable board and card games from natural language descriptions.

## Project Structure

```
app/              → React/Vite/Tailwind frontend (UI prototype)
  src/
    App.tsx       → Root layout: sidebar nav, header, AI chat bar, stage routing
    types.ts      → Core types: AppState, Stage, Card, CardType, BriefSettings, SimulationData
    constants.ts  → Design system tokens, stage definitions, taglines
    components/
      UI.tsx      → Shared components: Button, Card, Badge, AISuggestion, StatInput
    stages/       → One component per pipeline stage
      BriefStage.tsx    → Stage 1: Natural language input + AI concept generation
      DesignStage.tsx   → Stage 2: Card type editor + component manifest
      BalanceStage.tsx  → Stage 3: Simulation dashboard (Recharts)
      ArtStage.tsx      → Stage 4: Style guide + asset gallery
      LayoutStage.tsx   → Stage 5: Canvas card editor with drag zones
      ExportStage.tsx   → Stage 6: PDF, TTS, Game Crafter, crowdfunding kit
research/         → Market research, personas, competitive analysis, architecture docs
```

## Tech Stack

- **Framework**: React 19 + TypeScript 5.8
- **Build**: Vite 6
- **Styling**: Tailwind CSS v4 (uses `@theme` directive, NOT `tailwind.config.js`)
- **Animation**: Motion (Framer Motion v12+, imported from `motion/react`)
- **Charts**: Recharts 3
- **Icons**: Lucide React
- **Utilities**: clsx + tailwind-merge (combined in `cn()` helper in UI.tsx)

## Key Conventions

### Tailwind v4
This project uses Tailwind CSS v4 which has a different config approach than v3. Theme customization is done via `@theme` in `index.css`, not in a config file. Custom colors are defined as CSS custom properties:
- `--color-bg`, `--color-surface`, `--color-surface-light`
- `--color-accent`, `--color-accent-hover`, `--color-ai`
- Use these as Tailwind classes: `bg-bg`, `bg-surface`, `text-accent`, `text-ai`, etc.

### Component Patterns
- All shared UI components are in `components/UI.tsx` and use `forwardRef`
- Stage components receive `{ state: AppState; setState: Dispatch<SetStateAction<AppState>> }` as props
- AI-generated content uses the `ai` prop on Card or the `ai-border` / `ai-shimmer` CSS classes
- The `cn()` utility merges Tailwind classes safely — always use it for conditional classes

### State Management
- All app state lives in a single `AppState` object in `App.tsx` (no external state library)
- State is passed down via props, not context
- Stage components update state via `setState(prev => ({ ...prev, ... }))`

### Design System
- **Dark mode default**, light mode via `.light-mode` class on `:root`
- Accent color: warm amber/gold (`#F59E0B`)
- AI indicator: purple gradient (`#8B5CF6` → `#D946EF`)
- Spacing: 8px grid
- Border radius: sm=4px, md=8px, lg=12px, xl=24px
- Card type colors are in `DESIGN_SYSTEM.colors.cardTypes` array

### Keyboard Shortcuts
- `1`–`6`: Jump to pipeline stages
- `Alt+Left/Right`: Previous/next stage
- Shortcuts are disabled when focus is in input/textarea/select elements

## Guidelines

- The app is currently a **UI prototype** with mock data — no backend or AI integration yet
- When adding new stages or features, follow the existing pattern: one file per stage in `stages/`
- Keep the `AppState` type in `types.ts` as the single source of truth
- Don't add state management libraries — the current useState approach is intentional for this phase
- Placeholder images use `picsum.photos` — these will be replaced with real AI generation later
- Dependencies include `@google/genai`, `express`, `better-sqlite3` from the AI Studio scaffold — these are not actively used yet and can be removed or replaced when backend work begins
